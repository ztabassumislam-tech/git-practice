# git-practice
In this lab, we learned how to use Trello and GitHub for software project management and version control.Using Trello, we learned how to create a project board, add lists and cards, organize tasks, assign work to team members, set deadlines, and monitor project progress. This helped us understand how software projects are planned and managed in a team.We also learned the basics of Git and GitHub. We created a GitHub repository, added a C++ file, and made our first commit. Then we cloned the repository to our local computer using 'git clone'. We learned how to check the repository status using 'git status', edit project files, and compare changes using 'git diff'. After making changes, we staged the files using 'git add', saved the changes with a meaningful commit message using 'git commit', and uploaded the updated project to GitHub using 'git push'. We also learned how Git tracks project history, records different versions of files, and makes collaboration easier for multiple developers.Overall, this lab helped us understand how to manage software projects using Trello and how to use Git and GitHub for version control, source code management, and team collaboration. These are important tools used in real-world software development.
